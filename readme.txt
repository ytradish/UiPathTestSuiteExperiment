TestCase.xaml

このテストは次の処理を行います
・ブラウザ移動
・A-SaaSログインページへ遷移
・A-SaaSログイン処理
・A-SaaS会計仕訳画面起動
・A-SaaS会計仕訳対象月のバリデーション処理
・A-SaaS会計仕訳画面終了
・A-SaaSログアウト処理

課題
・A-SaaS会計仕訳画面起動がたまに失敗する(通知ポップアップにフォーカスを奪われることがたまにある)
・ログインとログアウトをライブラリ化するべき
・各処理の命名は自動生成されたものなので、ちゃんと命名する必要がある
・オブジェクトリポジトリでXPathを管理するべき
・タブ切り替え時に5secのDelayを入れているが、より安定した待機処理に変更するべき
・UiPath Assitantから実行できない

はまりどころ
・ブラウザでマルチタブ制御する場合はActivityを別々にする必要がある(キーボードショートカットでタブを切り替えてもアクティブなタブは変わらない)
・プロパティのクローズ動作は明示的に設定したほうがいい。タブ切り替え時にA-SaaSのルート画面が閉じられてしまう挙動があり、その回避方法のためクローズ動作をNeverに設定した。
